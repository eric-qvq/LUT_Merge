
# FPGA 双输出 LUT 逻辑映射优化工具

> 基于 Yosys 开源 EDA 框架，针对紫光同创 FPGA 平台的 LUT 合并优化工具。  
> 本项目为 **FPGA EDA 算法竞赛——"FPGA 双输出 LUT 的逻辑映射算法"** 的参赛作品。

---

## 项目背景

### 什么是 LUT？

在 FPGA 芯片中，**查找表（Look-Up Table, LUT）** 是最基本的逻辑单元。一个 N 输入的 LUT 本质上是一个 $2^N$ 行的真值表，可以实现任意 N 输入的布尔函数。

紫光同创 FPGA 提供以下 LUT 类型：

- **GTP_LUT1 ~ GTP_LUT6**：单输出查找表，分别有 1~6 个输入
- **GTP_LUT6D**：**双输出**查找表，有 6 个输入（I0~I5）和 2 个输出（Z 和 Z5）

### 赛题目标

将技术映射后的网表中的**多个小型 LUT 合并为双输出 GTP_LUT6D**，在保证逻辑功能不变的前提下减少 LUT 总数，降低芯片面积开销。

### 评分标准

评分使用 cost 函数，综合考虑：

- **num_of_luts**：LUT 单元总数
- **num_of_pins**：引脚总数
- **max_level**：最大逻辑深度

最终得分为优化后的 cost 与基准 cost 的比值（ratio），越低越好。多个测试用例取几何平均。

### 两种合并模式

#### 标准模式（Standard）

将两个 LUT 合并为一个 GTP_LUT6D，其中 I5 引脚接一个真实信号作为选择线：

- **Z5 输出**：当 I5=0 时的函数值（对应原 z5_lut 的功能）
- **Z 输出**：完整的 6 输入函数（对应原 z_lut 的功能）
- 约束：非常量输入并集 ≤ 6

#### 独立模式（Independent）

两个功能独立的 LUT 合并，I5 固定为常量 1：

- **Z5 输出**：由 INIT 低 32 位定义（I5=0 的那一半）
- **Z 输出**：由 INIT 高 32 位定义（I5=1 的那一半）
- 约束：非常量输入并集 ≤ 5

### 打分公式

每个候选合并对的得分：

$$\text{score} = W_{mode} + N_{shared} \times W_{shared} + N_{pins} \times W_{pins} - D_{penalty} \times W_{depth} - F_{penalty} \times W_{fanout}$$

其中：

| 符号          | 含义                  | 默认权重 |
| ------------- | --------------------- | -------- |
| $W_{mode}$    | 模式奖励（标准/独立） | 400 / 0  |
| $N_{shared}$  | 共享非常量输入数      | ×30      |
| $N_{pins}$    | 节省的引脚数          | ×6       |
| $D_{penalty}$ | 逻辑深度增加量        | ×80      |
| $F_{penalty}$ | 高扇出惩罚            | ×4       |

### 关键技术点

1. **迭代式 Kahn 拓扑排序**：用 BFS 替代递归 DFS 计算逻辑层级，避免深层电路的递归栈溢出
2. **增量式环路检测**：维护持久化信号图，每次合并只做局部 BFS 可达性判断，复杂度从 O(V+E) 降到 O(reachable)
3. **共享输入反向索引**：预建信号→LUT 映射，快速找到有共享输入的 LUT 对
4. **多进程并行候选搜索**：利用 ProcessPoolExecutor 加速大规模网表的候选生成
5. **多层正确性保障**：功能等价性穷举验证 → INIT 敏感性检查 → 增量环路检测 → 全图环路验证 → 不可变单元指纹校验

## 使用方法

### 环境依赖

- Python 3.8+
- [Yosys](https://github.com/YosysHQ/yosys) 开源综合工具（需编译安装含紫光 score 功能的版本）

### 基本用法

```bash
# 对单个网表进行 LUT 合并优化
python3 techlibs/pango/M_L_V05.py <输入网表.v> -o <输出网表.v> --yosys ./yosys

# 示例
python3 techlibs/pango/M_L_V05.py public_cases/design_1.v -o design_1_syn.v --yosys ./yosys
```

### 常用参数

| 参数                  | 说明                       | 默认值               |
| --------------------- | -------------------------- | -------------------- |
| `input`               | 输入映射后的 Verilog 网表  | （必须）             |
| `-o, --output`        | 输出文件路径               | `<输入文件名>_syn.v` |
| `--yosys`             | yosys 可执行文件路径       | 自动查找             |
| `--allow-independent` | 启用独立模式合并           | 关闭                 |
| `--merge-all-luts`    | 合并所有 LUT（不限制命名） | 仅合并 `*_lut`       |
| `--jobs`              | 并行进程数                 | 6                    |
| `--time-budget-sec`   | 超时限制（秒）             | 1680（28分钟）       |
| `--verify`            | 合并后用 yosys check 验证  | 关闭                 |
| `--protect-from`      | 指定参考网表保护不可变单元 | 无                   |

### 打分权重调整

```bash
python3 M_L_V05.py input.v \
  --w-shared 30 \        # 共享输入权重
  --w-pins 6 \           # 引脚节省权重
  --w-depth 80 \         # 深度惩罚权重
  --w-fanout 4 \         # 扇出惩罚权重
  --w-mode-standard 400  # 标准模式奖励
```

### 使用评测脚本

```bash
cd techlibs/pango/program
python3 run_designs.py ../your_test_script.sh -case_dir ./public_cases -result_dir ./testresult
```

## 核心算法文件说明（M_L_V05.py）

### 数据结构

- **`Lut`**：表示一个查找表单元（名称、位宽、输入信号、输出信号、真值表）
- **`Candidate`**：表示一个候选合并对（两个 LUT 名、合并模式、得分等）
- **`ScoreWeights`**：打分权重配置
- **`IncrementalCycleChecker`**：增量式环路检测器（维护信号前向图）

### 主要函数

| 函数                           | 功能                             |
| ------------------------------ | -------------------------------- |
| `extract_luts()`               | 从网表中提取所有可合并的 LUT     |
| `compute_lut_levels()`         | 用 Kahn 拓扑排序计算逻辑层级     |
| `compute_signal_fanout()`      | 统计每个信号的扇出数             |
| `can_merge_standard()`         | 判断两个 LUT 能否做标准模式合并  |
| `can_merge_independent()`      | 判断两个 LUT 能否做独立模式合并  |
| `estimate_candidate_metrics()` | 计算候选合并对的综合得分         |
| `build_candidates()`           | 多进程并行生成所有候选合并对     |
| `select_disjoint_pairs()`      | 贪心选择互不冲突的最优对         |
| `apply_merges()`               | 执行合并并做增量环路检测         |
| `build_init_standard()`        | 构造标准模式的 64 位 INIT 真值表 |
| `build_init_independent()`     | 构造独立模式的 64 位 INIT 真值表 |
| `detect_lut_cycles()`          | 全图 DFS 环路检测（安全后备）    |
| `repair_bad_lut6d_cells()`     | 修复错误合并产生的坏 LUT6D       |
